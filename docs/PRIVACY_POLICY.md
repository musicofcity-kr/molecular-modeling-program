# 개인정보처리방침

`다양한 분자의 분자구조 모델링`(이하 "본 서비스")은 개인정보 보호법 제30조에 따라 이용자의 개인정보를 보호하고, 개인정보 처리와 관련한 고충을 신속하고 원활하게 처리하기 위하여 다음과 같이 개인정보처리방침을 수립·공개합니다.

본 문서는 교사 개발 학습지원 소프트웨어 표준 개인정보처리방침 형식을 바탕으로 작성하였으며, 현재 웹앱의 실제 동작에 맞추어 학생 회원가입, 서버 제출 저장, Firestore 저장은 아직 운영하지 않는다는 점을 반영했습니다.

## 1. 웹사이트 게시 및 운영 안내

본 서비스는 웹앱 화면 하단에 다음 항목을 표시합니다.

- 개인정보처리방침
- 이용약관
- 정보관리책임자
- 저작권 표시

학생은 현재 회원가입 없이 수업코드, 입장 확인코드, 수업용 닉네임 또는 익명 ID로 활동에 입장합니다. 실명, 학번, 전화번호, 이메일, 주소 등 불필요한 개인정보는 입력하지 않도록 안내합니다.

## 제1조 개인정보의 처리 목적

본 서비스는 다음 목적을 위하여 필요한 최소 범위의 정보를 처리합니다. 처리 목적이 변경되는 경우에는 관련 법령에 따라 별도 안내 또는 동의 절차를 진행합니다.

1. 수업 활동 입장 및 진행
   - 수업코드를 통한 활동 구분
   - 수업용 닉네임 또는 익명 ID를 통한 현재 브라우저 세션의 활동 식별

2. 학습 활동 지원
   - 분자 구조 그리기, 구조 확인, 분자식 및 평균 분자량 표시
   - 3D 구조 보기, 활동 질문 응답, 활동 결과 정리

3. 임시 저장 및 내보내기
   - 학생이 선택한 경우 현재 브라우저에 활동 결과를 임시 저장
   - 보고서 저장, 인쇄 등 수업 활동 기록 생성

4. 교사용 기능 준비
   - Firebase Auth 기반 교사용 로그인 구조 준비
   - 향후 수업방 생성, 활동 관리, 제출 확인 기능을 위한 화면 및 서비스 계층 준비

## 제2조 개인정보의 처리 및 보유기간

현재 본 서비스는 학생 개인정보와 제출물을 서버에 저장하지 않습니다.

1. 학생 세션 정보
   - 처리 항목: 수업코드, 입장 확인코드, 수업용 닉네임 또는 익명 ID, 익명 학생 식별자, 활동 시작 시각
   - 보유 위치: 현재 브라우저의 앱 실행 상태
   - 보유 기간: 브라우저 새로고침, 탭 종료, 앱 상태 초기화 시 소멸

2. 활동 결과 임시 저장
   - 처리 항목: 활동 응답, 구조 확인 결과, 3D 관찰 기록, 정리 답변
   - 보유 위치: 학생이 사용하는 브라우저의 localStorage
   - 보유 기간: 이용자가 브라우저 데이터를 삭제하거나 앱에서 저장 결과를 삭제할 때까지

3. 서버 저장 기능
   - 현재 production 저장 기능은 비활성화되어 있습니다.
   - 향후 Firestore 저장 기능을 도입하는 경우, 보유 기간은 학년도 종료 시 또는 수업 운영 목적 달성 시까지로 별도 안내합니다.

## 제3조 처리하는 개인정보 항목

본 서비스는 현재 단계에서 서버로 학생 개인정보를 수집하지 않습니다. 다만 웹앱 사용 과정에서 다음 정보가 브라우저 내부에서 처리될 수 있습니다.

1. 학생 활동 정보
   - 수업코드
   - 수업용 닉네임 또는 익명 ID
   - 익명 학생 식별자
   - 활동 질문 응답
   - 학생이 그린 분자 구조 데이터
   - 구조 확인 결과와 활동 정리 내용

2. 교사용 인증 준비 정보
   - 현재 실제 인증은 실행하지 않습니다.
   - 향후 Firebase Auth 연결 시 교사 계정의 이메일, 표시 이름, 인증 UID 등이 처리될 수 있습니다.

3. 수집하지 않는 정보
   - 주민등록번호
   - 주소
   - 전화번호
   - 학생 이메일
   - 실명
   - 학번
   - 민감정보

## 제4조 만 14세 미만 아동의 개인정보 처리에 관한 사항

현재 본 서비스는 학생 회원가입과 서버 제출 저장을 운영하지 않습니다. 학교 수업에서 만 14세 미만 학생이 본 서비스를 사용하는 경우, 담당 교사는 학교의 개인정보 보호 지침과 가정통신문 등 필요한 절차를 확인해야 합니다.

향후 학생 계정 생성, 서버 제출 저장, 교사 피드백 저장 등 개인정보 처리 범위가 확대되는 경우에는 법정대리인 동의가 필요한지 검토하고 별도 동의 절차를 마련합니다.

## 제5조 개인정보의 파기 절차 및 방법

1. 서버 저장 정보
   - 현재 학생 개인정보와 제출물을 서버에 저장하지 않으므로 서버 DB 파기 대상은 없습니다.

2. 브라우저 임시 저장 정보
   - 활동 결과 임시 저장 데이터는 이용자가 브라우저 저장공간을 삭제하거나 앱에서 저장 결과를 삭제하면 제거됩니다.
   - 브라우저 저장소에 남아 있는 데이터는 해당 기기와 브라우저에서만 접근 가능합니다.

3. 향후 서버 저장 도입 시
   - 보유 기간 경과 또는 처리 목적 달성 후 지체 없이 파기합니다.
   - 전자적 파일은 복구할 수 없는 방식으로 삭제합니다.

## 제6조 개인정보의 안전성 확보조치

본 서비스는 개인정보 보호를 위하여 다음 조치를 원칙으로 합니다.

1. 최소 수집
   - 학생 실명, 학번, 연락처를 기본 입력 항목으로 요구하지 않습니다.

2. 서버 저장 제한
   - Firestore Security Rules 설계와 교사용 인증 구조가 확정되기 전까지 학생 제출 데이터를 production 서버에 저장하지 않습니다.

3. 환경변수 관리
   - Firebase 설정값과 배포 환경값은 `.env` 또는 배포 환경변수로 관리합니다.
   - service account, private token, 비공개 키는 공개 저장소에 저장하지 않습니다.

4. HTTPS 운영
   - 배포 시 HTTPS 기반 호스팅 환경을 사용합니다.

5. 접근 권한 분리
   - 학생 화면과 교사용 화면을 분리하고, 교사용 비공개 정보는 실제 인증 구조가 준비되기 전까지 학생에게 노출하지 않습니다.

## 제7조 정보주체와 법정대리인의 권리·의무 및 행사방법

학생 및 법정대리인은 개인정보 열람, 정정, 삭제, 처리정지를 요청할 수 있습니다.

현재 서버 저장 기능이 없으므로 학생이 브라우저에 임시 저장한 활동 결과는 해당 브라우저에서 직접 삭제할 수 있습니다. 학교 수업 운영과 관련한 문의는 담당 교사 또는 정보관리책임자에게 요청할 수 있습니다.

향후 서버 저장 기능이 도입되는 경우, 서비스 내 삭제 요청 또는 담당자 문의를 통해 필요한 조치를 진행합니다.

개인정보 보호를 위하여 교사의 개인 휴대전화 번호는 공개하지 않습니다.

## 제9조 개인정보처리방침 변경

이 개인정보처리방침은 2026년 7월 1일부터 적용됩니다.

서비스 구조가 변경되어 학생 계정, 교사 인증, Firestore 저장, 제출물 서버 저장, 외부 분석 연동 등이 활성화되는 경우 본 방침을 수정하고 웹앱 화면을 통해 안내합니다.
