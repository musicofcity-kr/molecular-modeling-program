## 2. 디자인 컨셉: "불꽃 레일 (Flame Rail)"

**한 문장 정의:** 학생이 단계를 완료할 때마다 진행 레일의 원소 노드가
해당 원소의 불꽃 반응색으로 "점화"되는, 화학 수업만이 가질 수 있는
진행 시스템.

**근거:** 기존 phase 색상(#a63347 진홍 / #12615f 청록 / #a66a08 주황 /
#5a3a87 보라)은 이미 불꽃 반응색 계열이다. 이 설계는 새 컨셉을 발명하는
것이 아니라 코드에 잠들어 있던 자산을 화면의 정체성으로 끌어올리는 것이다.
불꽃 반응은 고1 화학 교육과정의 대표 실험이므로 진행 UI 자체가 복습
장치가 된다.

| Phase | 원소 | 불꽃색 (신규 토큰) | 기존 색 (계승원) |
|---|---|---|---|
| 예측 (1–2단계) | Li 리튬 | `#D9435F` | #a63347 (명도 상향) |
| 구축 (3단계) | Cu 구리 | `#0F8F86` | #12615f (명도 상향) |
| 검증 (4–5단계) | Na 나트륨 | `#E8930C` | #a66a08 (채도 상향) |
| 성찰 (6–7단계) | K 칼륨 | `#7C5CC4` | #5a3a87 (명도 상향) |

**색만으로 상태를 구분하지 않는다 (색각 이상 대응):** 점화된 노드는
색 + 원소기호 + 체크 아이콘 3중 표기. 현재 노드는 색 + 이중 링 + 굵은 라벨.

  --surface: #FFFFFF;
  --surface-dim: #EEF2F8;

  /* 잉크 */
  --ink: #1B2430;
  --ink-soft: #5A6B7E;

  --phase-predict-soft: #FCEBEE;
  --phase-build-soft:   #E6F4F3;

  --radius-card: 16px;
  --radius-chip: 999px;

## 4. 구조 설계: "한 화면 = 한 단계" 위저드

현재 세로 나열식을 단계별 위저드로 전환한다. 데이터 구조(7단계, slot
props)는 그대로 두고 **렌더링만 조건부**로 바꾼다.

### 4.1 데스크톱 (≥1024px)

```
┌──────────────────────────────────────────────────────────┐
│ 헤더(56px): 활동명 · 수업방 이름          [저장됨 ✓] [나가기] │
├──────────────────────────────────────────────────────────┤
│ 불꽃 레일(88px):                                            │
│  (Li✓)──(Li✓)──(Cu✓)──【Na】──(Na)──(K)──(K)              │
│  예측        구축      ▲검증 4/7      성찰                  │
├──────────────────────────────────────────────────────────┤
│                                                            │
│        ┌────── 스테이지 카드 (max 880px) ──────┐            │
│        │ ▎phase색 상단 띠 4px                  │            │
│        │ 04 · 검증  구조 확인하기               │            │
│        │                                       │            │
│        │  [내 예측]        [확인 결과]          │            │
│        │  분자식 H2O   →   분자식 H2O  ✓ 일치   │            │
│        │  분자량 18    →   분자량 18.02 ✓ 일치  │            │
│        │                                       │            │
│        └───────────────────────────────────────┘            │
│                                                            │
├──────────────────────────────────────────────────────────┤
│ 액션 바(72px):  [← 이전]        [다음: 입체 구조 보기 →]     │
└──────────────────────────────────────────────────────────┘
```

- 스테이지 카드는 화면에 **항상 1개만**. 카드 상단에 현재 phase 색 4px 띠.
- 하단 액션 바는 고정(sticky). primary 버튼은 현재 phase 색, 라벨은
  반드시 "다음: {다음 단계 이름}" 형식 — 학생이 다음에 뭘 하는지 예고.
- 예외 — 3단계(구조 그리기): 스테이지 max-width를 1280px로 확장,
  Ketcher 최소 높이 560px. 그리기 공간이 최우선.
- 예외 — 5단계(입체 구조 보기): 예측 모형 / 참고 3D 를 좌우 2열 비교.

### 4.2 모바일 (<768px)

- 불꽃 레일은 축약형: 현재 원소 노드 1개 + "검증 4/7" 텍스트 +
  가는 진행 바. 탭하면 전체 레일이 바텀시트로 펼쳐짐.
- 스테이지 카드 좌우 여백 16px, 액션 바는 하단 고정.
- 3단계 Ketcher는 세로 최소 60vh.

### 4.3 검증 단계(4단계)의 판정 언어

RDKit 실측값과 학생 예측을 비교해 배지를 표시한다:
- 일치: `✓ 예측과 일치해요` (--ok)
- 불일치: `△ 예측과 달라요 — 어디가 다른지 살펴보세요` (--retry)

"오답/틀림" 어휘와 빨간색을 쓰지 않는다. 불일치는 탐구의 출발점으로
프레이밍한다 (예측→검증→성찰 교육과정 취지 부합).

### 4.4 모션

- 노드 점화: 완료 순간 해당 원소색으로 200ms 채움 + 1회성 glow
  (box-shadow 확산 400ms). 반복 애니메이션 없음.
- 단계 전환: 스테이지 카드 fade+slide 150ms.
- `prefers-reduced-motion: reduce` 시 모든 전환 즉시 적용.

## 5. 접근성·품질 바닥선

- 모든 상태는 색 + 문자/아이콘 이중 표기
- 키보드: 레일 노드 좌우 화살표 이동, 액션 바 버튼 focus ring 2px (phase 색)
- 대비: 본문 텍스트 대비비 ≥ 7:1 (--ink on --surface = 13.9:1 충족),
  phase 색 텍스트는 대면적 배경 위 사용 금지 (배지·띠·버튼에만)
- 터치 타깃 최소 44×44px

## 6. 이번 범위에서 하지 않는 것

- 교사 화면 리디자인 (별도 사이클)
- Ketcher 내부 툴바 커스터마이징 (라이브러리 영역)
- 다크 모드
- 일러스트/캐릭터 도입 (검토 후 별도 결정)
