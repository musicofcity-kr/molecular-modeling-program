# 공개형 분석기 재구축 마스터 체크리스트

## 작업 통제

- [ ] 기존 `apps/workbench` 수정 금지
- [ ] 작업 브랜치 확인
- [ ] 현재 Phase 번호 확인
- [ ] 해당 Phase의 스킬 파일 읽기
- [ ] Phase 범위를 벗어난 기능 추가 금지
- [ ] 새 의존성 추가 전 목적·라이선스·번들 영향 기록
- [ ] 구현 전 실패 테스트 또는 수동 재현 기준 작성
- [ ] 구현 후 최신 검증 명령 실행
- [ ] 완료 보고 작성

## Phase 진행표

| Phase | 이름 | 상태 | 차단 조건 |
|---|---|---|---|
| 0 | 기준선 동결 | 미시작 | 기존 앱 빌드 상태 미확인 |
| 1 | 제품 계약 | 미시작 | 범위·비목표 미확정 |
| 2 | 새 앱 스캐폴딩 | 미시작 | 기존 앱과 독립 빌드 불가 |
| 3 | Ketcher 입력 | 미시작 | 구조 추출 계약 불명확 |
| 4 | 화학 분석 코어 | 미시작 | 수용 테스트 실패 |
| 5 | 다중 입력 | 미시작 | 입력별 분석 계약 불일치 |
| 6 | 결과 UI | 미시작 | 출처·경고 표시 부족 |
| 7 | 3D 자료 | 미시작 | 구조 일치 확인 부족 |
| 8 | 제한적 VSEPR | 미시작 | 범용 모양으로 오인 가능 |
| 9 | 내보내기 | 미시작 | 출처·제한 메타데이터 누락 |
| 10 | 테스트 체계 | 미시작 | 핵심 RDKit 테스트 mock 처리 |
| 11 | 성능·접근성·보안 | 미시작 | 모바일·키보드 흐름 실패 |
| 12 | 배포 전환 | 미시작 | Preview·롤백 검증 미완료 |

## 매 작업 시작 전 질문

1. 이 변경은 현재 Phase의 목표에 직접 필요한가?
2. 화학값의 출처와 계산 방법이 명확한가?
3. 실패 상태가 성공 상태와 구분되는가?
4. 기존 앱을 건드리지 않고 구현할 수 있는가?
5. 테스트가 이 결함을 다시 잡을 수 있는가?

## 매 작업 종료 전 질문

1. 변경된 모든 파일을 설명할 수 있는가?
2. RDKit·PubChem·3Dmol·VSEPR의 역할을 혼합하지 않았는가?
3. 잘못된 구조에서 결과를 차단하거나 제한 표시하는가?
4. 최신 테스트를 실제로 실행했는가?
5. 완료 선언을 뒷받침하는 로그가 있는가?
